# 数据表字段说明文档

## 一、customer_base.csv - 客户基本信息表

**表说明**：存储客户的基础档案信息，包括个人属性、职业信息、收入水平和开户信息等静态数据。

### 字段列表

| 字段名 | 数据类型 | 业务含义 | 示例值 |
|--------|----------|----------|--------|
| customer_id | 字符串 | 客户唯一标识ID，用于关联其他业务表 | 316d72c0795f4fa9a61aeb8804c21b8f |
| name | 字符串 | 客户姓名 | 王丹 |
| age | 整数 | 客户年龄（岁） | 49 |
| gender | 字符串 | 客户性别 | 男/女 |
| occupation | 字符串 | 客户具体职业 | 律师、教师、医生、UI设计师等 |
| occupation_type | 字符串 | 职业分类类型 | 专业人士、事业单位、企业高管、传统行业、互联网从业者 |
| monthly_income | 浮点数 | 客户月收入（元） | 49458.85 |
| open_account_date | 日期 | 客户在银行开户的日期 | 2019-08-22 |
| lifecycle_stage | 字符串 | 客户生命周期阶段 | 新客户、成长客户、成熟客户、忠诚客户 |
| marriage_status | 字符串 | 婚姻状况 | 已婚/未婚 |
| city_level | 字符串 | 客户所在城市级别 | 一线城市、二线城市 |
| branch_name | 字符串 | 客户所属银行分支机构名称 | 招商银行上海分行外滩支行 |

### 字段详细说明

**lifecycle_stage（客户生命周期阶段）**：
- **新客户**：刚开户或开户时间较短的客户
- **成长客户**：与银行建立初步合作关系，业务量逐步增长的客户
- **成熟客户**：与银行保持稳定合作关系，业务量稳定的客户
- **忠诚客户**：长期与银行保持深度合作关系的高价值客户

**occupation_type（职业类型分类）**：
- **专业人士**：律师、医生等需要专业资质的职业
- **事业单位**：教师、医院职工等事业编制人员
- **企业高管**：企业CFO等高级管理人员
- **传统行业**：行政人员、制造业职工等
- **互联网从业者**：UI设计师等互联网相关职业

**表说明**：按月统计的客户行为和资产动态数据，记录客户的资产变化、产品持有情况、消费行为、APP使用行为和营销互动信息。同一客户每月生成一条记录。

### 字段详细说明

**资产相关字段**：
- **total_assets**：客户在本行的所有资产总和，包括存款、理财、基金、保险等
- **deposit_balance** ~ **insurance_balance**：各类产品的余额明细
- **asset_level**：根据总资产金额划分的客户等级，用于客户分层管理

**产品持有标志**：
- **deposit_flag/financial_flag/fund_flag/insurance_flag**：布尔标志（0或1），快速识别客户持有哪些类型的产品
- **product_count**：客户持有的不同产品数量，反映客户的产品组合丰富度

**行为指标**：
- **financial_repurchase_count**：理财产品复购次数，衡量客户对理财产品的忠诚度
- **investment_monthly_count**：当月投资操作次数，反映客户的投资活跃度
- **credit_card_monthly_expense**：信用卡消费金额，反映客户的消费能力和活跃度

**APP使用行为**：
- **app_login_count**：登录频次，反映客户的活跃程度
- **app_financial_view_time**：理财页面浏览时长，反映客户对理财产品的兴趣度
- **app_product_compare_count**：产品对比次数，反映客户的购买意向强度

**营销互动**：
- **last_contact_time**：记录最近一次与客户的营销联系时间
- **contact_result**：联系结果分类
  - **未接通**：客户未接听电话或未响应
  - **成功**：成功与客户沟通
  - **考虑中**：客户表示需要考虑
- **marketing_cool_period**：营销冷静期，避免过度打扰客户的保护机制

**时间维度**：
- **stat_month**：数据统计的月份，用于时间序列分析和趋势追踪

## 三、表关系说明

### 主键与外键关系

- **customer_base.customer_id**：客户基础信息表的主键
- **customer_behavior_assets.customer_id**：外键，关联到customer_base表
- **customer_behavior_assets.id**：行为资产表的主键（每条月度记录的唯一标识）

### 数据关联特点

1. **一对多关系**：一个客户（customer_base）对应多条月度行为记录（customer_behavior_assets）
2. **时间序列**：customer_behavior_assets按月记录，可追踪客户的资产和行为变化趋势
3. **数据更新频率**：
   - customer_base：相对静态，客户基本信息变化较少
   - customer_behavior_assets：动态更新，每月生成新记录

### 典型使用场景

- **客户画像分析**：结合两表数据，构建完整的客户360度视图
- **趋势分析**：通过behavior_assets的时间序列数据，分析客户资产增长、行为变化趋势
- **精准营销**：基于客户属性（base表）和行为特征（behavior_assets表）进行客户分群和营销策略制定
- **流失预警**：监控APP登录频次、资产变化等指标，识别潜在流失客户

## 四、数据质量说明

### 空值处理

- **last_contact_time** 和 **contact_result**：部分记录可能为空，表示该月未进行营销联系
- 其他数值字段：一般不应为空，为0表示无该项数据

### 数据时效性

- **stat_month**：标识数据的统计时间，确保数据分析时注意时间范围
- **last_app_login_time**：精确到秒，可用于实时性分析

### 建议使用注意事项

1. 分析客户行为趋势时，建议至少使用3个月以上的数据
2. 营销活动分析需结合**marketing_cool_period**，避免在冷静期内的数据误判
3. 资产分析时注意区分**总资产**和**各类产品余额**的关系
4. APP行为指标都是当月累计值，跨月对比时需注意时间周期一致性
